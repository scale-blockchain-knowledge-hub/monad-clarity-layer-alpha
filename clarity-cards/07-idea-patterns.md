# Idea Patterns for Monad Builders

These conceptual patterns help identify what to build on Monad.

### 1. High-Frequency Applications
- On-chain trading engines  
- Real-time scoreboards  
- Microtransaction-based games  

### 2. Multi-User Independent Workflows
- Marketplace matching  
- Auction rooms  
- Distributed collaboration apps  

### 3. State-Light, Compute-Heavy Apps
- Simulation logic  
- On-chain calculations  
- ZK-proof batching helpers  

### 4. Optimistic Interaction Flows
Where conflicts are rare or manageable.
